**ArenaWorldManager (AWM)** - это инфраструктурный плагин для Paper (1.21.1), предназначенный для детерминированного управления состоянием игровых арен. Он изолирует логику работы с миром (копирование, вставка схем, регионы) от игровой логики (волны, награды).

### Ключевые возможности:

* **Два метода восстановления:** Полное клонирование мира (`WORLD_CLONE`) и быстрая вставка через FAWE (`SCHEMATIC_PASTE`).
* **Умная очистка:** Асинхронный поиск и удаление сущностей по типам (Items, Mobs, Projectiles) без фризов сервера.
* **Интеграция с WorldGuard:** Автоматическое создание регионов и наложение флагов при подготовке арены.
* **Лимитирование нагрузки:** Встроенный семафор предотвращает перегрузку I/O при массовом ресете арен.

## Конфигурация карт (Templates)

Все шаблоны арен хранятся в папке `plugins/ArenaWorldManager/maps/`. Каждая карта - это отдельный `.yml` файл.

**Пример файла `forest_arena.yml`:**

```yaml
# Имя папки в корне сервера, которая служит эталоном
source-world: "templates/forest_v1"

# Метод: WORLD_CLONE (копия папки) или SCHEMATIC_PASTE (вставка .schem)
method: "WORLD_CLONE"

# Область, в которой будет производиться очистка сущностей и наложение регионов
cleanup-bounds:
  x1: -100
  y1: 60
  z1: -100
  x2: 100
  y2: 120
  z2: 100

# Политика очистки (что удалять при подготовке/команде cleanup)
cleanup-policy:
  items: true
  mobs: true
  projectiles: true
  vehicles: false

# Автоматические регионы WorldGuard
regions:
  - id: "arena_zone"
    priority: 10
    flags:
      pvp: "allow"
      mob-spawning: "deny"
      greeting: "Вы вошли на арену!"

```

## Автоматическое определение карт

Плагин поддерживает **Zero-Config** добавление небольших арен через схематики. Вам не обязательно писать YAML-файл для каждой новой карты.

**Как это работает:**
1. Поместите ваш файл схематики (например, `colosseum.schem`) в папку `plugins/ArenaWorldManager/schematics/`.
2. Перезапустите сервер.
3. Плагин автоматически прочитает размеры постройки и создаст виртуальный конфиг с именем `auto_colosseum`.

**Особенности авто-карт:**
* При подготовке (`/awm prepare`) постройка будет вставлена ровно по центру в координатах `X:0, Y:100, Z:0`.
* Плагин сам вычислит размер постройки из файла. Команда очистки (`/awm cleanup`) будет удалять мусор только строго внутри стен арены.
* В командах просто указывайте префикс: `/awm prepare match_01 auto_colosseum`.

*Примечание:* Если вам нужно добавить кастомные регионы WorldGuard или изменить политику очистки для сгенерированного конфига карты, просто создайте YAML-файл с таким же именем (`auto_colosseum.yml`), и плагин отдаст приоритет ручным настройкам.

## Командный интерфейс

Для работы команд требуется право `arena.worldmanager.admin`.

| Команда | Описание | Аргументы |
| --- | --- | --- |
| `/awm prepare` | Создает/восстанавливает арену | `<arenaId> <templateId>` |
| `/awm reset` | Удаляет мир арены и файлы | `<arenaId>` |
| `/awm cleanup` | Чистит сущности по границам | `<arenaId>` |
| `/awm reload` | Перезагружает конфиги карт | — |

## Руководство для разработчика (API)

Если вы пишете свой плагин, используйте API для управления мирами.

### Получение доступа:

```java
ArenaWorldAPI api = (ArenaWorldAPI) Bukkit.getPluginManager().getPlugin("ArenaWorldManager");
```

### Пример подготовки арены:

```java
api.prepareArena("match_01", mapConfig).thenAccept(v -> {
    // Этот код выполнится, когда мир скопирован, 
    // сущности удалены и регионы созданы.
    Bukkit.getLogger().info("Арена готова к приему игроков!");
    
    // Телепортация игроков и старт таймера
    startMatch("match_01");
});
```

### Пример удаления арены:

```java
worldApi.resetArena(arenaId).thenRun(() -> {
    // Этот код выполнится, когда сущности и мир будут удалены.
    Bukkit.getLogger().info("Арена " + arenaId + " успешно удалена и очищена.");
});
```

### Получение конфигурации карты:

```java
// Получаем API AWM (через сервис или напрямую из плагина)
ArenaWorldAPI api = arenaWorldManagerPlugin.getApi();

String templateId = "auto_colosseum";
String arenaInstanceId = "match_01";

api.getMapConfig(templateId).ifPresentOrElse(config -> {
        // Если конфиг найден, готовим арену
        api.prepareArena(arenaInstanceId, config).thenRun(() -> {
            Bukkit.getLogger().info("Арена " + arenaInstanceId + " готова!");
        });
    }, () -> {
        // Если конфиг не найден (например, файл удалили или переименовали)
        Bukkit.getLogger().warning("Ошибка: Конфиг " + templateId + " не найден в реестре!");
});
```

## Обработка ошибок

Плагин следует принципу **Fail-Closed**:

1. Если при копировании возникла ошибка (нет места на диске) — `CompletableFuture` завершится с исключением.
2. Арена **не будет** помечена как готовая.
3. В логах и в чате админа появится детальное сообщение об ошибке.
   Это предотвращает ситуации, когда игроки попадают на "сломанную" или недостроенную карту.
